2. Manipulación I: filtrar y seleccionar 🔍¶

Idea clave: filtrar = quedarse con FILAS que cumplan condición. Seleccionar = quedarse con COLUMNAS específicas.

In [3]:
# Filtrar filas con .query()  — LEGIBLE
ventas_q4 = df.query("quarter == 'Q4'")
print(f'Filas en Q4: {len(ventas_q4):,}')

# Filtrar filas con .loc[] — más flexible
ventas_lima = df.loc[df['region'] == 'Lima']
print(f'Filas en Lima: {len(ventas_lima):,}')

# Combinar condiciones — usar paréntesis y operadores & | (no and/or)
ventas_q4_lima = df.loc[(df['region'] == 'Lima') & (df['quarter'] == 'Q4')]
print(f'Filas en Lima Q4: {len(ventas_q4_lima):,}')

Filas en Q4: 1,277
Filas en Lima: 860
Filas en Lima Q4: 229

⚠️ Real-World Warning: &/| no and/or¶

Cuando combinas condiciones en pandas, usa & (and) y | (or) — no and/or que dan error.

Y SIEMPRE pon paréntesis alrededor de cada condición:

# ✅ Correcto
df.loc[(df['x'] > 5) & (df['y'] < 10)]

# ❌ Error
df.loc[df['x'] > 5 and df['y'] < 10]

💡 Pro-Tip: .agg() con sintaxis named-tuples¶

La sintaxis nombre=('columna', 'operación') es la moderna (pandas ≥ 0.25). Es legible, permite múltiples agregaciones de la misma columna, y nombra las salidas.

Operaciones comunes: 'sum', 'mean', 'median', 'count', 'nunique', 'std', 'min', 'max', 'first', 'last'.

5. ⚠️ Trampa #4: Vectorización implícita ⚡¶

En R: vec1 * vec2 opera elemento-a-elemento sin pensar. Y si los tamaños no coinciden, R recicla silenciosamente.

En Python con listas nativas: la multiplicación NO funciona elemento-a-elemento.

En Python con numpy: funciona elemento-a-elemento, pero da ERROR si los tamaños no coinciden (no recicla).

In [12]:
# Listas nativas: multiplicación NO es elemento-a-elemento
lista_a = [1, 2, 3]
lista_b = [10, 20, 30]

try:
    resultado = lista_a * lista_b
except TypeError as e:
    print(f'❌ Error: {e}')
    print('Las listas Python no soportan operaciones vectoriales.')

❌ Error: can't multiply sequence by non-int of type 'list'
Las listas Python no soportan operaciones vectoriales.
In [13]:
# Con numpy: SÍ es elemento-a-elemento
arr_a = np.array([1, 2, 3])
arr_b = np.array([10, 20, 30])

print(f'arr_a * arr_b = {arr_a * arr_b}')   # [10, 40, 90]

arr_a * arr_b = [10 40 90]
In [14]:
# Tamaños distintos: numpy ERROR, R hubiera reciclado
arr_grande = np.array([1, 2, 3, 4, 5, 6])
arr_chico = np.array([10, 20])

try:
    resultado = arr_grande * arr_chico
except ValueError as e:
    print(f'❌ numpy error: {e}')
    print('🔁 En R esto hubiera devuelto silenciosamente: [10, 40, 30, 80, 50, 120]')

❌ numpy error: operands could not be broadcast together with shapes (6,) (2,) 
🔁 En R esto hubiera devuelto silenciosamente: [10, 40, 30, 80, 50, 120]

⚠️ Real-World Warning¶

Si portas código de R a Python esperando reciclaje, vas a ver ValueError. Esto es bueno: numpy te protege de un bug silencioso. R te lo deja pasar con un warning que es fácil ignorar.

💡 Pro-Tip: matplotlib vs seaborn¶

  • matplotlib (plt.bar, df.plot): control fino, más verboso. Bueno para reportes.
  • seaborn (sns.heatmap, sns.barplot): atajos para gráficos estadísticos comunes. Bueno para EDA rápida.

Ambos coexisten — seaborn está construido sobre matplotlib.

# ============================================================
# PIPELINE: Top 5 productos por ingresos Q4 por región
# ============================================================

def construir_pipeline_top_productos(df: pd.DataFrame,
                                      quarter_objetivo: str = 'Q4',
                                      top_n: int = 5) -> pd.DataFrame:
    """Pipeline reproducible — toma df y devuelve top N productos por región."""
    return (
        df
        # Paso 1: filtrar quarter objetivo
        .query(f"quarter == '{quarter_objetivo}'")
        # Paso 2: limpiar — eliminar registros con cantidad sospechosa (>=1000)
        .loc[lambda d: d['cantidad'] < 1000]
        # Paso 3: agrupar por región × producto, sumar ingresos
        .groupby(['region', 'producto'], as_index=False)
        .agg(ingresos_totales=('ingresos', 'sum'),
             unidades_vendidas=('cantidad', 'sum'))
        # Paso 4: ranking dentro de cada región
        .assign(rank=lambda d: d.groupby('region')['ingresos_totales']
                                .rank(method='dense', ascending=False))
        # Paso 5: quedarse con top N
        .query(f'rank <= {top_n}')
        .sort_values(['region', 'rank'])
        .reset_index(drop=True)
    )

# Ejecutar
top_productos = construir_pipeline_top_productos(df, quarter_objetivo='Q4', top_n=5)
print(f'Resultado: {len(top_productos)} filas (6 regiones × 5 productos = 30)')
